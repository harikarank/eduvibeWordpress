## Changes
- Registered CPT `course` in `functions.php` with slug `courses`, full Gutenberg support, custom icon
- Registered taxonomy `course_category` (hierarchical) attached to `course` CPT
- Added `nla_course_meta($key, $default='')` helper function in `functions.php`
- Created `archive-course.php`: breadcrumb, 3-column course grid with `.card-hover-action` overlay, `edu-pagination`
- Created `single-course.php`: breadcrumb trail (Home → Courses → Title), col-xl-8 main content + col-xl-4 enroll sidebar
- Course card fallback thumbnail: `assets/images/course/course-01/course-01.jpg`
- Homepage (`front-page.php`): WP_Query for latest 10 courses rendered in Slick carousel

## Impact
- After activation, flush WP permalinks: Admin → Settings → Permalinks → Save
- Course archive available at `/courses/`
- Requires at least 1 published Course post for homepage carousel to render
- Enroll Now button in single sidebar links to `#` — needs wiring to payment/LMS plugin
- Wishlist button on cards is UI-only (no backend)
- Star rating in archive cards is static (5 stars) — dynamic rating needs a plugin

## Validation
- CPT registration verified against `course.html` card structure (class `edu-card card-type-3 radius-small`)
- Category badge `eduvibe-status status-03` confirmed correct (blog uses `status-05` — different)
- `nla_course_meta()` uses `get_post_meta()` with `esc_html()` — XSS safe
- Taxonomy slug `course-category` confirmed not conflicting with any standard WP taxonomy

## Rollback
- Remove `nla_register_cpt_course()` and `nla_register_course_taxonomy()` from `functions.php`
- Delete `archive-course.php` and `single-course.php` from theme folder
- Flush permalinks after removal
- No course posts or taxonomy terms deleted automatically — data remains in DB
