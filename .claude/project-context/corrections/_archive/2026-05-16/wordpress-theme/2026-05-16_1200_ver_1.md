## Changes
- Initial conversion of 7 static HTML pages into a complete WordPress theme
- Created `next-level-academy/` theme folder with all required template files
- `header.php`: logo, desktop nav with `NLA_Nav_Walker`, mobile popup menu — shared across all pages
- `footer.php`: 4-column footer (logo+social, Explore, Useful Links, Contact Info), copyright, `wp_footer()`
- `front-page.php`: hero banner, services (4 cards), courses Slick carousel (dynamic CPT query), events section, testimonials slider, why-choose-us, newsletter form, latest 3 blog posts
- `archive-course.php`: course grid 3-col, hover card overlay, `edu-pagination` pagination
- `single-course.php`: breadcrumb, col-xl-8 main + col-xl-4 enroll sidebar
- `archive.php`: blog grid 3-col, breadcrumb, pagination
- `single.php`: full-width centered blog post (col-lg-10 offset-lg-1), tags, prev/next
- `page-about.php`: template "About Us" — about image gallery, stats, services grid
- `page-contact.php`: template "Contact Us" — info cards, CF7 form, Google Map
- `page.php` and `index.php`: generic fallbacks
- `functions.php`: theme setup, asset enqueueing, CPT/taxonomy registration, `NLA_Nav_Walker`, `nla_course_meta()` helper
- `style.css`: WordPress theme header + CF7 scoped CSS overrides
- `assets/` folder copied into theme root (mirrors original HTML asset paths)

## Impact
- Replaces static HTML site entirely
- All internal links updated from `.html` extensions to WordPress permalink functions (`home_url()`, `get_permalink()`, `get_post_type_archive_link()`)
- Navigation driven by `wp_nav_menu()` — menus must be assigned in WP Admin → Appearance → Menus
- Page templates must be assigned in WP Admin → Pages → Page Attributes → Template

## Validation
- Verified against all 7 source HTML files: `index.html`, `course.html`, `course-details.html`, `blog.html`, `blog-details.html`, `about-us.html`, `contact-us.html`
- Header/footer HTML structure matches original exactly (classes, mobile menu, back-to-top SVG)
- Asset enqueue order matches original `<script>` load order in HTML files
- `NLA_Nav_Walker` confirmed to output `has-droupdown` and `submenu` class names matching original HTML nav

## Rollback
- Delete `next-level-academy/` theme folder from `wp-content/themes/`
- Revert to original static HTML files in repo root
- No database changes made by the theme itself (CPT registration is reversible via `functions.php` removal)
