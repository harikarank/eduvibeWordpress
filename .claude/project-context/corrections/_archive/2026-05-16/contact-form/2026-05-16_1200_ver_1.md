## Changes
- Replaced native fallback form in `page-contact.php` with Contact Form 7 integration
- CF7 shortcode `[contact-form-7 title="Contact Form"]` rendered inside `.rnt-contact-form.rwt-dynamic-form` wrapper
- Removed `if ( function_exists('wpcf7') )` check — CF7 is assumed always active
- Added CF7 admin setup instructions as PHP comment in `page-contact.php`
- Added ~100 lines of scoped CSS to `style.css` to override all CF7 default styles:
  - `.wpcf7-form-control-wrap` → block with 20px bottom margin (simulates `.form-group`)
  - `input[type="text/email/tel"]` → `height:60px`, `bg:#F5F5F5`, `border-radius:5px`, `padding:0 30px`
  - `textarea` → `min-height:180px`, `padding:17px 30px`, same background
  - Focus → `border-color: var(--color-primary)`
  - `.wpcf7-submit` → full `edu-btn` style (primary colour, 60px height, 100% width, bold)
  - `.wpcf7-spinner` → `display:none`
  - `.wpcf7-not-valid` → red border `#e74c3c`
  - `.wpcf7-not-valid-tip` → 13px red text below field
  - `.wpcf7-response-output` → green banner on success, red on failure/invalid/spam

## Impact
- Visual design of contact form is unchanged from original HTML
- CF7 handles submission, validation, and email routing — no custom PHP handler needed
- All CF7 CSS scoped to `.rnt-contact-form` — zero impact on any other element on the site
- CF7 form must be created manually in WP Admin with title "Contact Form" and correct field tags

## Validation
- CSS values (`#F5F5F5`, `border-radius:5px`, `height:60px`, `padding:0 30px`) extracted directly from `.rwt-dynamic-form` rules at `style.css:12966–12995`
- `edu-btn` values (`height:60px`, `background: var(--color-primary)`, `font-size:18px`, `font-weight:700`) extracted from `style.css:6598–6611`
- Scoped selector `.rnt-contact-form` confirmed present on the wrapper div in `page-contact.php`

## Rollback

- In `style.css`: remove the CF7 override CSS block (everything after the theme header comment)
